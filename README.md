
# Filmstrip - Global Publishing Instructions

Follow these exact steps to move from **Simulation** to **Production**.

### 1. The Database (Supabase)
Go to your Supabase SQL Editor and paste this to create your tables:

```sql
-- 1. Create Users table
create table public.users (
  id uuid references auth.users not null primary key,
  email text,
  "firstName" text,
  "lastName" text,
  name text,
  username text unique,
  "avatarUrl" text,
  "favoriteGenres" text[],
  "favoriteMovieId" text,
  "createdAt" timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. Create Entries table
create table public.entries (
  id bigint generated by default as identity primary key,
  "userId" uuid references auth.users not null,
  "movieId" text not null,
  status text check (status in ('WATCHED', 'WATCH_LATER', 'DROPPED')),
  rating int,
  "droppedReason" text,
  notes text,
  timestamp bigint
);

-- 3. Create Challenges table
create table public.challenges (
  id uuid default gen_random_uuid() primary key,
  "creatorId" uuid references auth.users,
  "recipientId" uuid references auth.users,
  "turnUserId" uuid references auth.users,
  type text,
  size int,
  status text,
  "movieIds" text[],
  results jsonb,
  timestamp bigint
);
```

### 2. The Hosting (Vercel)
1. Import your project to Vercel.
2. Add these Environment Variables:
   - `VITE_SUPABASE_URL`: Your project URL.
   - `VITE_SUPABASE_ANON_KEY`: Your "anon public" key.
   - `API_KEY`: Your Google Gemini API key.

### 3. Verification Emails
By default, Supabase sends real emails from its own server (limited to 3 per hour on free tier).
To send thousands of emails, go to **Authentication -> Email Templates** in Supabase and connect a service like **Resend** or **SendGrid** in the "SMTP" settings.

### 4. Global URL
Once Vercel finishes deploying, they will give you a link like `filmstrip.vercel.app`. That is your live, global web app!
